Component Styles Reference

Complete guide for creating custom component templates

Available Template Variables

Top-Level Variables

Variable Type Description
content String/HTML The main content to be styled
anchor String Optional anchor/heading text
Custom Variables Various Any variables defined in the style's "variables" section

Basic Component Template

<div class="component">
  {{#anchor}}
    <h2 class="heading">{{anchor}}</h2>
  {{/anchor}}
  
  <div class="content">
    {{{content}}}
  </div>
</div>
Important: Use triple braces {{{content}}} to render HTML content without escaping!

Custom Variables

You can define custom variables in your component style's "variables" section:

{
  "borderColor": {
    "type": "color",
    "default": "#e2e8f0"
  },
  "padding": {
    "type": "text",
    "default": "2rem"
  },
  "showIcon": {
    "type": "boolean",
    "default": true
  }
}

Then use them in your template:

<div class="component" style="border-color: {{borderColor}}; padding: {{padding}};">
  {{#showIcon}}
    <span class="icon">✨</span>
  {{/showIcon}}
  {{{content}}}
</div>

Example: Card Component

Template:

<div class="card">
  {{#anchor}}
    <div class="card-header">
      <h3>{{anchor}}</h3>
    </div>
  {{/anchor}}
  
  <div class="card-body">
    {{{content}}}
  </div>
  
  {{#showFooter}}
    <div class="card-footer">
      <button>Learn More</button>
    </div>
  {{/showFooter}}
</div>

CSS:

.card {
  background: white;
  border-radius: 8px;
  box-shadow: 0 2px 8px rgba(0,0,0,0.1);
  overflow: hidden;
}
.card-header {
  background: {{headerBg}};
  color: {{headerColor}};
  padding: 1.5rem;
  border-bottom: 2px solid {{borderColor}};
}
.card-header h3 {
  margin: 0;
  font-size: 1.5rem;
}
.card-body {
  padding: 1.5rem;
}
.card-footer {
  padding: 1rem 1.5rem;
  background: #f7fafc;
  border-top: 1px solid #e2e8f0;
  text-align: right;
}
.card-footer button {
  background: {{buttonBg}};
  color: white;
  border: none;
  padding: 0.5rem 1.5rem;
  border-radius: 4px;
  cursor: pointer;
}

Variables:

{
  "headerBg": {
    "type": "color",
    "default": "#667eea"
  },
  "headerColor": {
    "type": "color",
    "default": "#ffffff"
  },
  "borderColor": {
    "type": "color",
    "default": "#e2e8f0"
  },
  "buttonBg": {
    "type": "color",
    "default": "#667eea"
  },
  "showFooter": {
    "type": "boolean",
    "default": true
  }
}

Example: Callout/Alert Component

Template:

  <div class="callout-icon">
    {{#icon}}{{icon}}{{/icon}}
    {{^icon}}ℹ️{{/icon}}
  </div>
  
  <div class="callout-content">
    {{#anchor}}
      <div class="callout-title">{{anchor}}</div>
    {{/anchor}}
    <div class="callout-message">

Mustache Syntax Reference

Syntax Purpose Example
{{variable}} Escaped output {{anchor}}
{{{variable}}} Unescaped HTML {{{content}}}
{{#variable}} Section (if true) {{#anchor}}...{{/anchor}}
{{^variable}} Inverted section (if false) {{^icon}}Default{{/icon}}
{{!comment}} Comment {{! This is a comment }}

Best Practices

  • Always use {{{content}}} (triple braces) for HTML content
  • Provide fallback content for optional variables
  • Use semantic HTML elements
  • Make components responsive with CSS
  • Test with different content lengths
  • Use CSS custom properties for theming
  • Ensure accessibility (ARIA labels, semantic markup)
  • Keep templates simple and reusable
